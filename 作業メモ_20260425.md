### 切り替え：face-api.js版（完全無料・ブラウザ内処理）
- `frontend/src/pages/FaceHappinessPage.tsx` を全面書き直し
- CDN（jsdelivr）からface-api.jsとモデルを動的ロード
- 写真はサーバーに送信せずブラウザ内で完結
- 判定ロジック（expressionToHappiness関数）
  - happy > 0.5 → レベル5（とても幸せ）
  - happy > 0.2 → レベル4（幸せ）

  - negativeScore > 0.5 → レベル1（元気なし）
  - negativeScore > 0.2 → レベル2（少し元気なし）
  - それ以外 → レベル3（ふつう）

- 「ホームに戻る」ボタンも追加

## 3. 出版社モード（B案）の実装

### 方針
- ユーザーテーブルに `plan` カラム（standard / publisher）を追加
- 出版社ユーザーはホーム画面が「自分史＋顔幸福度チェック」のみに絞られる
- 将来複数の出版社に展開しやすいよう本体に組み込む設計（B案）

### DBマイグレーション
- `backend/src/routes/auth.ts` に `initPlanColumn()` を追加
- サーバー起動時に自動で `ALTER TABLE users ADD COLUMN IF NOT EXISTS plan VARCHAR(20) DEFAULT 'standard'` を実行

### バックエンド変更
- ファイル：`backend/src/routes/auth.ts`
- 登録API：`plan` パラメータを受け取り保存（publisher/standard のみ許可）
- ログインAPI：レスポンスに `plan` を含めるよう修正
- `/me` エンドポイント：`plan` カラムも返すよう修正

### フロントエンド変更
- `frontend/src/pages/FieldSelectPage.tsx`
  - `user.plan === 'publisher'` のとき出版社モードを表示
  - 出版社モード：自分史カードのみ＋顔幸福度チェックボタン
  - ヘッダータイトルを「自分史アプリ」に変更
  - 設定ボタンを非表示

### 専用登録URL（案B）
- `frontend/src/pages/RegisterPage.tsx` を修正
- URLパラメータ `?plan=publisher` を読み取り登録時にDBへ保存
- 出版社モード時の登録画面の変更点：
  - 「📖 自分史アプリ 出版社プラン」バッジを表示
  - 記録の種類選択を非表示（自分史固定）
  - 登録完了後は団体コード画面をスキップしてホームへ直行

### 出版社への案内URL
```
https://memo.robostudy.jp/register?plan=publisher
```
このURLを出版社に送るだけで出版社モードで登録できる。
